### **Esercizio 3: Restyle sito web**

**Modernizzazione di un sito web datato – Full Stack Exercise** 

### **Situazione**

Un cliente vi contatta perché possiede un sito web **molto datato** (struttura HTML obsoleta, design anni 2000/2010, non responsive, non accessibile, performance scarse). Vi chiede di **modernizzarlo completamente** rispettando gli standard attuali di web design, accessibilità, usabilità e performance (2025/2026).

Il sito originale è descritto nel documento **“Spiegazione del layout”** e visibile nello **screenshot allegato** .

 

### **Obiettivi del Progetto e Punteggi**

| Punti | Fase | Descrizione | Skill acquisita |
| ----- | ----- | ----- | ----- |
| **20** | **1\. Ricreazione fedele del layout originale** | Ricreare **esattamente** il layout del sito originale utilizzando solo **HTML5 semantico** (senza CSS avanzati o framework). L’obiettivo è allenare la capacità di strutturare correttamente una pagina. | Struttura HTML semantica |
| **30** | **2\. Analisi e Progettazione UI/UX moderna** | Analizzare criticamente il sito esistente e proporre una versione aggiornata e moderna attraverso **Figma**: • **Wireframe** del sito aggiornato (10 pt) • **Mockup definitivo** (High-Fidelity) (10 pt) • **Design System** completo (font, palette colori, icone, componenti riutilizzabili, griglia, regole di spacing) (10 pt) | Prototipazione UI/UX, Design System, Figma |
| **50** | **3\. Sviluppo Front-End Moderno** | Implementare la versione moderna del sito utilizzando tecnologie di nuova generazione: • Framework CSS consigliati: **Tailwind CSS** (preferito) o **Bootstrap 5** • Sito **100% responsive** (mobile-first) • Codice pulito, accessibile (ARIA), performante e semanticamente corretto • Possibilità di aggiungere micro-interazioni o animazioni leggere | Sviluppo Front-End con Framework moderni |
| **Bonus** | **4\. Documentazione Tecnica** | Redigere un **documento tecnico professionale** (PDF o Markdown) che descriva tutte le fasi del progetto (analisi, scelte progettuali, problemi incontrati, soluzioni adottate, codice principale, conclusioni). | Technical Writing / Documentazione di progetto |

**Punteggio totale: 100 punti**

 

### **Requisiti Tecnici Obbligatori**

* **HTML5** semantico e valido (W3C)  
* **Mobile-First** e fully responsive  
* Accessibilità base (contrasti, heading corretti, alt text, focus visibili)  
* Ottimizzazione immagini e performance (Lighthouse score ≥ 90 consigliato)  
* Repository Git con commit chiari e messaggio descrittivo  
* Cartella progetto organizzata chiaramente (/src, /public, /docs, ecc.)

 

### **Consegna Finale**

Devi consegnare **una sola cartella ZIP** (o repository GitHub privato con link) contenente:

1. Cartella 01-original-html → versione fedele solo HTML  
2. Cartella 02-figma → file .fig (wireframe \+ mockup \+ design system)  
3. Cartella 03-sito-moderno → progetto completo con Tailwind/Bootstrap (index.html \+ assets)  
4. File DOCUMENTAZIONE-TECNICA.pdf o README.md dettagliato
